Skills are organized into bucket folders under `skills/`:

- `engineering/` — daily code work
- `productivity/` — daily non-code workflow tools
- `misc/` — kept around but rarely used
- `personal/` — tied to my own setup, not promoted
- `in-progress/` — drafts not yet ready to ship
- `deprecated/` — no longer used

Every skill in `engineering/`, `productivity/`, or `misc/` must have a reference in the top-level `README.md` and entries in all three platform manifests:

- `.claude-plugin/plugin.json` — Claude Code (generated by `scripts/gen-plugin-json.sh`)
- `antigravity.json` — Antigravity (generated by `scripts/gen-antigravity-json.sh`)
- `AGENTS.md` — OpenAI Codex CLI (generated by `scripts/gen-agents-md.sh`)

All three manifests are generated from the skills on disk — never edit them by hand. After adding or editing a skill, run:

```bash
./scripts/sync-manifests.sh
```

This regenerates all three manifests and warns about any shippable skill missing a `README.md` link. A `pre-commit` hook (in `.githooks/`) runs it automatically and stages the regenerated manifests on every commit that touches a skill, so they never drift. Enable the hook once per clone with:

```bash
git config core.hooksPath .githooks
```

Skills in `personal/`, `in-progress/`, and `deprecated/` must not appear in any of these.

Each skill entry in the top-level `README.md` must link the skill name to its `SKILL.md`.

Each bucket folder has a `README.md` that lists every skill in the bucket with a one-line description, with the skill name linked to its `SKILL.md`.
